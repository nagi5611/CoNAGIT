# ファイルマッピングシステム 実装用プロンプト

以下の仕様に従い、**親オブジェクト（プロジェクト）ごとのファイルマッピング機能**を、この機能を持っていない既存アプリに実装してください。既存の「プロジェクト」「子プロジェクト」「ファイル」「ユーザー」のデータモデルはそのまま利用し、地図上にスポットを配置して各スポットにファイルを割り当てる機能を追加します。

---

## 1. 前提とする既存モデル

- **プロジェクト (projects)** … 親オブジェクト。id, name 等を持つ。
- **子プロジェクト (subprojects)** … プロジェクトに属する。project_id で紐づく。
- **ファイル (files)** … 子プロジェクトに属する。subproject_id で紐づく。file_type = 'file' のレコードが「ファイル」、'folder' はフォルダ。
- **ユーザー (users)** … id, username 等。

※ テーブル名・カラム名は自アプリに合わせて読み替えてよい。リレーション（親→子プロジェクト→ファイル）が同じであればよい。

---

## 2. 追加するデータベース

### 2.1 map_spots（スポット）

地図上の1つのポイント。プロジェクト単位で管理する。

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | UUID 等 |
| project_id | TEXT NOT NULL | プロジェクトID（FK） |
| floor | INTEGER NOT NULL | 階。1, 2, 3 など（地図画像の種類に対応） |
| name | TEXT NOT NULL | スポット名 |
| x_percent | REAL NOT NULL | 地図画像内の横位置（0〜100） |
| y_percent | REAL NOT NULL | 地図画像内の縦位置（0〜100） |
| display_order | INTEGER | 表示順（任意） |
| created_at | DATETIME | 作成日時 |
| created_by | TEXT | 作成者ユーザーID（FK、任意。ツールチップ用） |

- 座標は**必ず画像に対する相対値（パーセント）**で保存する。ピクセルではなく % にすることで、PC・スマホで表示サイズが変わっても位置がずれないようにする。
- project_id は ON DELETE CASCADE 推奨。

### 2.2 map_spot_files（スポットとファイルの多対多）

1スポットに複数ファイルを割り当て可能にする。

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INTEGER PK など | 連番 or UUID |
| spot_id | TEXT NOT NULL | map_spots.id（FK, CASCADE） |
| file_id | TEXT NOT NULL | files.id（FK, CASCADE） |
| display_order | INTEGER | 表示順（任意） |

## 3. 追加するAPI

ベースパスは自アプリに合わせて変更してよい。以下は `project_id` を `:id` で受ける想定。

1. **GET /api/projects/:id/files**  
   そのプロジェクトに属する**全子プロジェクト**配下のファイル（file_type = 'file'）を一覧で返す。  
   レスポンスに `subproject_id`, `subproject_name`（または相当）を含め、どの子のファイルか分かるようにする。

2. **GET /api/projects/:id/map-spots**  
   そのプロジェクトのスポット一覧。クエリ `?floor=1` で階でフィルタ可能。  
   作成者名を表示する場合は users と JOIN して `created_by_name`（username）を返す。

3. **POST /api/projects/:id/map-spots**  
   スポット作成。body: `{ name, floor, x_percent, y_percent, userId? }`。  
   `userId` があれば `created_by` に保存。

4. **PUT /api/projects/:id/map-spots/:spotId**  
   スポット更新。body: `{ name?, floor?, x_percent?, y_percent?, display_order? }`。  
   対象スポットが当該 project_id に属することを必ず検証する。

5. **DELETE /api/projects/:id/map-spots/:spotId**  
   スポット削除。紐づく map_spot_files も削除（CASCADE または明示削除）。  
   対象が当該 project_id に属することを検証する。

6. **GET /api/projects/:id/map-spots/:spotId/files**  
   そのスポットに紐づくファイル一覧。file_id, file_name, subproject_name 等を返す。

7. **PUT /api/projects/:id/map-spots/:spotId/files**  
   スポットへのファイル割り当てをまとめて置き換え。body: `{ fileIds: string[] }`。  
   fileIds に含まれるファイルがすべて**そのプロジェクト配下の子プロジェクトに属する**ことを検証し、他プロジェクトの file_id が含まれていたら 400 を返す。

---

## 4. 地図画像の扱い

- 階（floor）ごとに1枚の地図画像を用意する（例: F1, F2, F3 で3枚）。
- 画像は静的ファイル（例: `/static/img/floormap_1f.png` など）でよい。
- フロントでは floor の値から表示する画像URLを決める（例: 1 → floormap_1f.png）。

---

## 5. UIの要件

### 5.1 入口

- プロジェクト詳細画面（親オブジェクトの詳細）に「ファイルマッピング」ボタンまたはリンクを設け、クリックでファイルマッピング画面に遷移する。

### 5.2 ファイルマッピング画面の構成

- **階の切り替え**  
  F1 / F2 / F3 などのタブまたはドロップダウンで、表示する地図画像を切り替える。

- **地図エリア**  
  - 地図画像を表示する。  
  - **座標ずれ防止**: 地図を包むコンテナに **min-height などを付けない**。コンテナは画像の表示サイズと完全に一致させる（inline-block + 画像のみで高さを決める）。スポット用のオーバーレイも同じコンテナ内で `position: absolute; inset: 0` とし、画像と同一の座標系にする。  
  - クリック用レイヤーとマーカー用レイヤーは**別レイヤー**にし、**マーカーがクリックを受け取るレイヤーより手前に来る**ようにする（既存スポットをクリックしたときに新規作成されないようにする）。  
  - 新規スポット追加: 地図の空いている部分をクリックしたとき、クリック位置を**オーバーレイ要素の getBoundingClientRect()** でパーセントに変換し、その座標でスポット作成APIを呼ぶ。名前入力はモーダル等で行う。

- **スポットマーカー**  
  - 既存スポットは `left: x_percent%; top: y_percent%` で配置。マーカーは中央がその座標になるよう、幅・高さの半分だけ負の margin でずらす。  
  - **モバイルではアイコンを小さく**する（例: デフォルト 20px、md 以上で 32px）。サイズと margin だけをレスポンシブで変え、**left/top の % は変えない**ことで位置はずらさない。  
  - マーカーにカーソルを合わせたとき、地図上に**小さなポップアップ（ツールチップ）**を表示する。表示内容: **スポット名・作成日時・作成者名**。created_by が無い既存データは「作成者: -」などでよい。

- **右側（または下）のスポット情報パネル**  
  - 何も選択していないときは「マーカーをクリックするとスポット情報が表示されます」などのメッセージ。  
  - スポットのマーカーをクリックしたとき、そのスポットの情報を表示する:  
    - スポット名（インライン編集可とするとよい）  
    - 割り当てられたファイル一覧（子プロジェクト名 / ファイル名）  
    - 「ファイルを割り当て」ボタン（クリックでファイル選択UIを開く）  
    - 削除ボタン・選択解除ボタン  
  - ファイル割り当てUIでは、**そのプロジェクト配下の全子プロジェクトのファイル**を一覧表示し、チェックボックスで単一・複数選択可能にする。保存時に PUT .../map-spots/:spotId/files に fileIds を送る。

- **割り当て画面・割り当てファイル一覧でのプレビュー**  
  - ファイル一覧の各行にプレビューボタン（目のアイコン等）を付ける。  
  - 画像: プレビューURLを取得してモーダルで表示。HEIC の場合は必要に応じて変換表示。  
  - PDF: プレビューURLを新しいタブで開く。  
  - その他: プレビューURLを新しいタブで開く、またはダウンロードで確認。

- **戻る**  
  プロジェクト詳細に戻る操作を必ず用意する。

### 5.3 位置ずれを防ぐための注意

- スポット座標は**常に地図画像に対する 0〜100 のパーセント**で保存・表示する。  
- 地図コンテナは**画像の表示サイズと一致**させ、min-height で余白を作らない。  
- クリック位置の % 算出には、**クリックを受け取るオーバーレイ要素の getBoundingClientRect()** を使う（画像要素ではなく、オーバーレイ基準にするとレイアウト差の影響を減らせる）。  
- マーカーは「left/top を % で指定 + 幅・高さの半分の負マージンで中央揃え」とし、レスポンシブでは**マーカーサイズとマージンのみ変更**し、left/top の % は変えない。

## 6. 実装時のチェックリスト

- [ ] map_spots / map_spot_files のマイグレーション（および既存スキーマの更新）
- [ ] 上記7本のAPIを実装し、project_id と spot の所属チェックを行う
- [ ] プロジェクト詳細からファイルマッピング画面への入口
- [ ] 階切り替えと地図画像の表示
- [ ] 地図クリックで新規スポット作成（座標は %、コンテナは画像サイズに一致）
- [ ] マーカー表示（% 配置、モバイルでアイコン小さく）
- [ ] マーカークリックで右（または下）パネルにスポット情報表示
- [ ] スポット名・割り当てファイル一覧・ファイル割り当て・削除
- [ ] ホバーでツールチップ（スポット名・作成日時・作成者名）
- [ ] 割り当て画面・割り当て一覧でのファイルプレビュー
- [ ] 既存スポットクリックで新規作成されない（レイヤー順序の確認）

- 既存の「プロジェクト」「子プロジェクト」「ファイル」「ユーザー」のテーブル名・APIパスを書いておくと、そのまま読み替えて実装しやすい。
